# Recommended Information Architecture

**Platform**: AI-Native Project & Event Operations Platform

## Structure

- **Public/Marketing**:
  - Website: Home (value prop for events/projects), Features (by area: planning, execution, finance, AI), Use Cases (events, conferences, productions, logistics), Pricing, Blog.
  - SEO/AEO/GEO: Target "event operations software", "project event management platform", "AI for event planning".

- **Docs/User**:
  - docs/guides/ : User guides for running events/projects.
  - docs/volumes/ : Knowledge base (as current, focused on ops).
  - Self-host guide.

- **Developer**:
  - docs/api/ : API reference, SDKs, examples.
  - Integrations.

- **Internal/Strategy**:
  - docs/strategy/ : All brand, business, product strategy (this folder).
  - docs/layers/ : Engineering SSOT.
  - docs/architecture/ : ADRs.

## Principles

- Keep focused on ops (not generic OS).
- AI-optimized: Structured, searchable, RAG-friendly.
- Clear separation: Public (benefits) vs Internal (how).

See the research report for sources (e.g., Stripe, Supabase docs patterns).
