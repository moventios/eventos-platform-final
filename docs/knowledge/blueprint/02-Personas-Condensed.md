# Phase 2: Personas (Condensed Actionable Cards)

All personas built on ICP (mid-large hybrid ops orgs). Each includes the required fields + "How the platform (workflow-first) serves them" + AI opps + "Now/Next priority for their primary flows".

## 1. Owner / Executive Sponsor
**Goals:** Scale profitably, reduce operational risk & surprises, demonstrate maturity to board/partners/clients.
**Daily/Weekly:** 10-20 min overview of portfolio health (pipeline, utilization, margin, risk alerts).
**Pains:** Late discovery of margin erosion or capacity conflicts; fragmented reporting for audits.
**Authority:** Strategic & large financial approvals.
**KPIs:** Portfolio margin, repeat/expansion revenue %, audit compliance score, ops efficiency (rev/FTE).
**Info Needs:** Executive dashboard (aggregate + drill), exception alerts, scenario modeling.
**Platform Service:** One place to see "is the machine running well?" without opening 5 tools.
**AI:** "What is the projected Q3 margin if we win these 4 pending proposals?" "Top 3 risks this month and recommended mitigations."
**Priority:** Now — excellent command center + alerts.

## 2. Operations Director
**Goals:** Predictable delivery across all active work.
**Daily:** Review all projects/events, clear approval queue, resolve resource conflicts, coach managers.
**Pains:** Context loss, manual chasing for status, firefighting instead of steering.
**KPIs:** On-time %, average issue resolution time, team utilization balance.
**AI:** Conflict resolver, capacity forecasting, "Which events are at risk this week?"
**Priority:** Now (unified views + approvals).

## 3. Event / Project Manager (Primary Daily User)
**Goals:** Deliver excellent experience on time/budget; look professional to client.
**Daily Tasks (80% of time in tool):** Timeline management, task assignment, vendor coordination, client updates, change handling.
**Pains:** Version hell, hidden conflicts, chasing approvals, end-of-day status scramble.
**KPIs:** Milestone hit rate, client NPS per project, budget variance.
**Usage:** Lives in Project/Event workspace (Overview → Timeline → Tasks → Resources → Approvals → Activity).
**AI Opportunities:** Auto-draft run-of-show, risk flagging on timeline changes, "Rewrite this client update professionally".
**Priority:** Now + Next (core workflows).

## 4. Venue / Facility Manager
**Goals:** Maximize utilization safely, protect the physical asset.
**Daily:** Calendar of spaces, incoming requests, setup/teardown coordination, issue logging.
**Pains:** Overlaps, last-minute changes without visibility, manual access lists.
**KPIs:** Occupancy rate, incident count, setup accuracy.
**Platform:** Resources section + Booking flow + Access management.
**AI:** "Suggest optimal packing of 3 events in Hall A this weekend."

## 5. Sales / Account Manager
**Goals:** Close more, at better margins, with realistic promises.
**Daily:** Pipeline, proposal creation, client relationship.
**Pains:** Lack of real availability when promising dates; slow turnaround on custom quotes.
**AI:** Fast proposal drafts from brief + historical close data.

## 6. Finance
**Goals:** Accurate cash position, clean books, fast close, audit readiness.
**Pains:** Manual reconciliation, missing documentation for revenue recognition.
**Platform:** Auto-ledger from ops actions + immutable history + one-click close reports.
**Priority:** Next (full finance flow integration).

## 7. Procurement / Warehouse / Logistics
**Goals:** Right gear/crew at right place/time at right cost.
**Flows:** PO creation from plan, receiving, returns, damage logging.
**AI:** "Predict gear needs for next 6 weeks based on confirmed events."

## 8. Crew Lead / On-site Team
**Goals:** Clear instructions, easy check-in/check-out, ability to report issues fast.
**UI:** Mobile-first, minimal, task list + scan for access + incident button.
**AI:** "What's the fastest way to reroute setup given current delay?"

## 9. Vendor (External Portal / Limited)
**Goals:** Know when/where/what is expected, easy confirmation & change comms, get paid on time.
**Flows:** Receive assignment, confirm, upload docs, see payment status.

## 10. Client / Stakeholder
**Goals:** Visibility without micromanaging, easy approvals, beautiful experience.
**Portal:** Read-only + lightweight approvals + file sharing + progress view. Never full ops tool.

## 11-13. Marketing/Production Coordinator, Visitor/Attendee (limited access pass + schedule view), Compliance/Auditor (read-only immutable history view).

**Common Requirements Across Personas:**
- Role-based default landing + filters ("show only my venues").
- Strong mobile for field roles.
- Consistent approval surface.
- Context everywhere ("this change affects X other things — here's impact").
- AI always explains + requires explicit apply.

Full journey maps + "A day in the life" stories per persona live in expanded research docs.
